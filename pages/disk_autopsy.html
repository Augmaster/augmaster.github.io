Disk Analysis & Autopsy

Category: Digital Forensics | Difficulty: Medium | Date: Jan 06, 2025

Investigate a Windows 10 disk image using Autopsy to uncover key forensic artifacts.

Introduction

In this challenge, we used Autopsy to analyze a Windows 10 disk image. The goal was to extract forensic artifacts, including system information, user accounts, network details, and potential security threats.

Step 1: Loading the Disk Image

The forensic image was loaded into Autopsy using the provided .aut case file. After ensuring the correct disk image path, we examined key system files to extract details. 

            
              MD5 Hash: 3f08c518adb3b5c1359849657a9b2079

Step 2: System Information

Extracting system metadata, we found:

  • Computer Name: DESKTOP-0R59DJ3
  • IP Address: 192.168.130.216
  • MAC Address: 08-00-27-2c-c4-b9
  • Network Adapter: Intel(R) PRO/1000 MT Desktop Adapter

Step 3: User Accounts & Activity

We identified all user accounts present on the system: 

            
              H4S4N, joshwa, keshav, sandhya, shreya, sivapriya, srini, suba

The last logged-in user was: sivapriya

Step 4: Suspicious Activity

A flagged YARA rule file led to the discovery of a hacking tool: 

            
              Author: Benjamin DELPY (gentilkiwi)
              Tools Found: Lazagne, Mimikatz

A user also attempted to exploit a domain controller using an archive named: 2.2.0 20200918 Zerologon encrypted.zip

Conclusion

This challenge reinforced essential digital forensics skills, including:

  • Using Autopsy to extract and analyze disk artifacts
  • Identifying suspicious user activity and security threats
  • Extracting and verifying system and network information
  • Detecting hacking tools and unauthorized privilege escalation attempts
